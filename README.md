## My engineering philosophy

I believe AI can help engineers build faster, but architecture still decides whether the system can survive long term.

Some principles I use:

- Facts belong to Readers
- Rules belong to Engines
- State and execution belong to Applications
- Observability should be first-class
- Workflow is more important than isolated features
- Capabilities compound better than one-off functions
- AI should accelerate engineering without destroying boundaries
